Getting started

Download MeteorJS

Start downloading METEOR framework for your platform at https://www.meteor.com/install

Start the application

Run the following command in the meteor project root folder
meteor

Open your browser

If eveything goes fine, your application should be ready at http://localhost:3000/

 

General tips

  • Do not start from scratch, use an existing asset and modify it to learn how it works.
  • Explore the sources for ideas and sample code.
  • Use Firebug or Chrome Developer Tools to find bugs on your website. Using one of those tools will help you to save time analyzing the site and finding elements structure, like classes, id or tags
  • Quick tip: open your site with Chrome, press F12 and go to console tab, reload your page and if something goes wrong you will see your page errors in red text.
  • In case of error messages, someone might have seen it too, so you can try a Google search for a quick fix.

Structure 

 
We have followed the structure described in this tutorial
Anyway, it's not exacly the same as MeteorJs doesn't have a strict folder structure.
 
+---client
|   +---angle
|   |   +---js
|   |   |   +---modules
|   |   +---less
|   |       +---app
|   |       +---bootstrap
|   |       +---themes
|   +---lib
|   |   +---js
|   +---vendor
|   +---views
|       +---partials
+---private
|   +---i18n
+---public
|   +---img
|   +---vendor
+---server
 
client: contains the modules loaded in the client. 
 
  - angle: contains the main theme files for used in the Angularjs Seed project using the same structure for Javascript and Less styles
 
  - lib: contains the script that are loaded before other scripts. Here is the main module declaration
 
  - vendor: contains the vendor files loaded when the application start. Not lazy loaded.
 
  - views: contains the HTML views used in the application
 
private: this folder is only accesible from the server.
 
puiblic: accesible from the client side. It can contain the the main assets like images, fonts, etc
 
  - vendor: This is folder is special because here is where we place vendor assets that are loaded using the ocLazyLoad module
 
server: contains the script that runs only in the server.

Modules

 
The application is started using the angular bootstrap method in the file app.module.js.
This files contains also the main module declaration
 
 
(function() {
    'use strict';
    angular
        .module('angle', [
            'app.core',
            'app.routes',
            'app.sidebar',
            'app.navsearch',
            'app.preloader',
            'app.loadingbar',
            'app.translate',
            'app.settings',
            'app.utils'
        ]);
   
    if (Meteor.isCordova)
      angular.element(document).on("deviceready", onReady);
    else
      angular.element(document).ready(onReady);
    
    function onReady() {
      angular.bootstrap(document, ['angle']);
    }

})();
 
Modules organization
 
Within folder client\lib live also the rest of modules declaration. They are located under this lib folder so meteor loads them before the rest of the JS because we need to declare first our modules and then use them. 
 
Any other module declaration must be placed here and the rest of files (config, controllers, etc) should be placed whing the client/angle/js folder with their respective folder name.

Routes

 
Routes are defined in the following script and uses the Iron Router package.
Due to the location of the file all routes here works for client and server side, if you need only cliente side routes, just add them in a new file within the cliente folder.
 
/lib/routes.js
 
Router.map(function(){

  this.route('singleview', {
    path: '/'
  });

  this.route('submenu', {
    path: '/sub/menu'
  });

});
 
 
Reminder: Meteor has its  own script injection system. Because of that you don't have add your own script tags. All scripts in each folder are automatically added by Meteor into the app.
 

Templates

jQuery's dom.ready doesn't work as usual in a METEORJS templates scenario. Becuase of that we've built a simple snippet that will collect all module functions to be executed when templates are ready, similar to a dom.ready handler but can be executed manually later.
 
/client/lib/app.init.js defines the module handler
/lib/rendered.js  is where App.run() is executed to catch the ready state of the main layout
 
Usage example:
 
(function(window, document, $, undefined){

  App.ready(function(){

    // Your code here

  });

})(window, document, window.jQuery);
 
The rendered callback
 
Optionally you can use the Template rendered callback method to run javascript for each template.
 
Example file /client/templates/singleview.js
 
Template.singleview.rendered = function(){

  // Code for the singleview template

};
